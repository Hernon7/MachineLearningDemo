Session Tracking

The first two parts of this series focused on the basics: creating basic webpages and HTML templates. Let’s get started on something a little more interesting.

If you want to make a page that involves recurrent user input, you need to keep track of every user’s state somehow. This would allow data to be stored throughout a user’s session on the web application. The stored data serves as “memory” holding whatever’s happened beforehand.

To keep track of the preceding data, the use of “session” offers temporary storage linked to a particular user’s recent session. Unlike cookies, “session” data is stored on the server. A session is the time interval between when a user logs into a sever and when the user logs out.

In Flask, the “session” of client-side data stores as a cookie on their browser. One incredibly helpful feature of Flask is the possibility of importing sessions. This uses the session object in a similar process to a dictionary that is linked to every website user. Also, the session object can store data relating to a user, which remains even after browser crash or page refresh.

Using Forms to Post Information in Flask- Introduction to Flask-WTF

To handle web forms in Flask, Flask provides an extension called the Flask-WTF extension. This extension nicely integrates well with Flask. The Flask ecosystem features more extensions, which are very vital parts of Flask ability to solve a variety of problems. Flask extensions are installed using the regular pip in Python. To get started let’s install Flask-WTF:

pip install flask-wtf

The Flask-WTF extension represents web forms using Python classes. A form-class defines the form fields as class variables. Let’s get started by creating a new python script called “appforms.py”, which would allow us to set a user login form that:

Receiving Form Data and Enhancing Field Validation

After running the code, pressing the submit button would lead to a “Method Not Allowed” error.

The error is because the “appforms.py” serving as the Login view function can display the form, but it has no logic to process the submitted data yet. Attaching field validators to form fields avert invalid data from being acknowledged. This application’s approach to dealing with invalid data is by re-loading the form page, to allow a user to enter correct data input.

If you attempt to submit invalid data into the form, you’ll notice a similar validation mechanism.
